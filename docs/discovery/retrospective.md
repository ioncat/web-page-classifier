# Ретроспектива: итеративное развитие url-parser

| # | Этап | Проблема / Мотив | Что добавили | Результат |
|---|------|-----------------|--------------|-----------|
| 1 | **Хранилище** | CSV-файлы теряют прогресс при сбое, нет дедупликации | SQLite + `pending/done/error` статусы, `INSERT OR IGNORE` | Идемпотентный пайплайн — повторный запуск безопасен |
| 2 | **Пайплайн** | Ручная обработка URL | step1 (импорт) → step2 (title) → step3 (classify), каждый шаг изолирован | Можно запускать шаги независимо, прерывать и продолжать |
| 3 | **Классификация** | Нет автоматической категоризации | Ollama + `tags`-справочник подсказок, авто-синхронизация тегов | Локальная LLM без внешних API; категории становятся согласованнее по ходу прогона |
| 4 | **Выбор модели** | Непонятно какая модель лучше для конкретного корпуса | `--compare-models` → `model_results` (изолированно), side-by-side таблица, `--accept-model` | Можно сравнить N моделей без риска испортить основные данные |
| 5 | **Производительность** | 1 поток + 1 URL/запрос → GPU утилизация 5–10% | `--batch N` (нумерованный список в одном промпте) + `--workers N` (ThreadPoolExecutor) | 80–90% GPU при `--batch 10 --workers 4` |
| 6 | **Надёжность** | Thinking-модели тратят все токены на рассуждения и возвращают пустой ответ | `--no-think` (`think: false` в Ollama options) | qwen3, deepseek-r1 работают корректно |
| 7 | **Конфигурация** | Константы разбросаны по 5 файлам, редактирование = риск сломать код | `config/settings.py` + `config/prompts.py` — единая точка правки | Пользователь меняет параметры и промпты не касаясь логики |
| 8 | **Качество тегов** | Справочник накопил 288 мусорных записей (URL как теги, числа, предложения) | `_is_valid_tag()` фильтр + переработка промпта (rule-based + few-shot, 1 категория вместо 1–5 тегов) | Чистый справочник → модель видит осмысленные подсказки → точнее |
| 9 | **Наблюдаемость** | Нет способа быстро проверить состояние / оценить промпт без записи в БД | `--stats`, `--dry-run` + таймер + авто-лог в `dryrun_log.csv`, `temperature=0.0` | Итерация промпта без риска загрязнить данные; детерминированный вывод |

---

**Сквозная логика:** каждый этап решал одну конкретную боль предыдущего.
CSV → SQLite дала checkpointing. SQLite дала основу для сравнения моделей. Сравнение показало что модели расходятся → нужно качество тегов. Качество тегов потребовало наблюдаемости без побочных эффектов → dry-run.
