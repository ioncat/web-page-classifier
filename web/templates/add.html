Обработка новых

Полный цикл для URL со статусом pending: сначала парсинг (HTTP-запрос за title и description), затем классификация через LLM (Ollama).

Модель — какую LLM использовать. batch — сколько URL отправлять в один запрос к LLM (экономит токены, но снижает точность). workers — параллельные потоки (URL распределяются по доменам, чтобы не перегружать один сайт).

Повтор ошибок

URL, у которых парсинг завершился ошибкой (403, 404, 500, таймаут, сетевой сбой), сбрасываются в pending и проходят полный цикл заново: парсинг + классификация.

Повторить все — сбрасывает все ошибки без разбора. Только временные — повторяет только 5xx, 429 и сетевые ошибки; постоянные (404, 403, 410) остаются нетронутыми — повторять их обычно бессмысленно.

Дополучить описания

Для URL, которые уже успешно обработаны (done), но у которых пустой title или description. Делает повторный HTTP-запрос за метаданными — без LLM, только сетевой запрос. Категория и статус не меняются.

Рекомендуется workers=4–8 — чисто сетевая операция, LLM не задействован.
